# Odin Project: To-Do List App
> Todo lists are a staple in beginning webdev tutorials, the implementation can be basic. There is, however, a lot of room for improvement and many features that can be added.

## Minimum Requirements
- [ ] Seperate DOM and application logic into different modules.
- [ ] Have a To-Do object with the following fields and methods:
    - [ ] title
    - [ ] description
    - [ ] dueDate
    - [ ] priority
    - [ ] delete(): deletes a to-do.
- [ ] Have a Project object to store a list of to-dos.
- [ ] The User Interface should have the following views:
    - [ ] View all projects.
    - [ ] View all to-dos in all projects.
    - [ ] Single to-do expanded view. Allows user to edit details.
- [ ] Store user data in localStorage. 
    - [ ] Save new to-dos/projects in local storage.
    - [ ] Load stored user data when the app is first loaded.
